External JavaScript

External JavaScript refers to JavaScript code that is stored in a separate file and linked to an HTML document using the <script> tag. This approach offers several advantages, including:

  • Separation of Concerns: Keeping JavaScript code separate from HTML makes it easier to manage and maintain.
  • Code Reusability: You can reuse the same JavaScript file across multiple HTML pages.
  • Improved Performance: Browser caching can improve performance by storing JavaScript files locally after they have been loaded once.
  • Organization: External JavaScript files can be organized into folders and directories, making it easier to structure larger projects.

Linking External JavaScript File

To link an external JavaScript file to an HTML document, use the <script> tag with the src attribute. The src attribute specifies the path to the external JavaScript file.

 

        <script src="script.js"></script>

Ensure that the src attribute points to the correct location of the JavaScript file relative to the HTML document. You can include the <script> tag in the <head> or <body> section of your HTML document.

Example

Here's an example of an external JavaScript file named script.js:


        // script.js
        alert('Hello, this is an external JavaScript file!');

And here's how you link this external JavaScript file to an HTML document:


        <!DOCTYPE html>
        <html lang="en">
        <head>
            <meta charset="UTF-8">
            <meta name="viewport" content="width=device-width, initial-scale=1.0">
            <title>External JavaScript Example</title>
            <script src="script.js"></script>
        </head>
        <body>
            <h1>External JavaScript Example</h1>
        </body>
        </html>

When you open the HTML document in a web browser, it will display an alert dialog with the message "Hello, this is an external JavaScript file!"

Using external JavaScript files is a best practice for building scalable and maintainable web applications.
